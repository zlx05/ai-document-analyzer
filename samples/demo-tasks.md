# 演示用高层任务

这份任务清单不要当成最终 todo，而是当成项目里的高层 Epic。把它们批量导入任务后，再让 Agent 根据知识库继续拆解。

## 建议导入前端的高层任务

1. 接入真实 LLM 并优化 Agent 工具调用
2. 将前端重构为 React 工程
3. 增加 PDF 和 docx 文件解析能力
4. 完成 Docker 部署和上线文档
5. 整理项目架构图和面试讲稿

## Agent 适合输出什么

Agent 不应该只是重复这些任务，而应该进一步生成：

- 每个 Epic 下面的具体子任务
- 哪些任务适合今天完成
- 哪些任务依赖数据库、前端或后端改动
- 哪些任务适合写进简历亮点
- 哪些任务有风险，比如 API key、文件解析、部署环境
- 两周开发计划
- README Roadmap
- 面试讲稿

## 推荐问 Agent 的 Prompt

请基于知识库和当前 5 个高层任务，把“接入真实 LLM 并优化 Agent 工具调用”拆成 8 个可执行子任务，并说明每个任务要改哪些文件。

请检查当前任务列表，指出哪些任务太大，需要继续拆分，并自动创建更细的待办任务。

请根据知识库和任务列表，生成一个两周开发计划，按优先级排序。

请从面试角度分析这个项目还缺哪些亮点，并创建下一步任务。
